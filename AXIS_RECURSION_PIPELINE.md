# AXIS RECURSION PIPELINE

Axis evolves through continuity-preserving recursion cycles.

Each cycle must deepen deterministic infrastructure without increasing visible
surface complexity.

## Continuity Receipts

Every meaningful architecture pass should leave a continuity receipt:

- Goal.
- Files changed.
- Primitive boundaries preserved.
- Systems intentionally not touched.
- Verification run.

Continuity receipts prevent silent drift.

## Recursion Cycle Law

Future recursion cycles must preserve:

- Shell stability.
- Rail continuity.
- STO source of truth.
- Immutable event chronology.
- SigmaMemoryPackage runtime boundary.
- Canonical narrativeSeed vocabulary.
- Provider abstraction boundaries.
- Upload, auth, Supabase, Mux, replay, voice, and AI utility infrastructure.

## Multi-Model Workflow

Models may assist different parts of the workflow:

- Codex implements and stabilizes repo architecture.
- OpenAI synthesizes runtime context.
- DeepSeek may critique or reason within defined contracts.
- Perplexity may ground outside-world research with citations.

No model owns Axis truth.

Axis owns chronology and continuity.

## Forward Gravity

Every cycle must make the product feel:

- Calmer.
- Lighter.
- More inevitable.
- More memory-native.
- More deterministic underneath.

Avoid cycles that create:

- Dashboard energy.
- Visible orchestration.
- Feature sprawl.
- Provider exposure.
- Speculative architecture masquerading as implemented primitives.

## Primitive Boundaries

Implemented primitives are load-bearing.

Research metaphors remain non-load-bearing until formalized into deterministic
code and contracts.

No silent promotion from metaphor to architecture.
