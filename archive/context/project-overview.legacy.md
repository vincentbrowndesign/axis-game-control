# Project Overview

Axis is an athletic continuity operating system.

The stabilized product direction is:

identity -> participation -> progression -> history -> leaderboard -> return behavior

Current core loop:

show up -> participate -> progression updates -> history grows -> leaderboard updates -> return tomorrow

Axis is not a media engine right now. Future layers can only sit on top of persistent identity, participation, and history.

## Core Purpose

- Give each member a persistent Axis identity.
- Turn participation into visible progression.
- Save check-ins, check-outs, completed hours, streaks, sessions, organization activity, and leaderboard placement into Axis History.
- Make consistency feel earned, remembered, socially alive, and worth returning to.
- Give organizations a trustworthy operating layer for culture, effort, and accountability.

## Emotional Primitive

Axis should feel like keeping a save file alive.

A member shows up, participates, completes effort, sees history grow, understands where they stand, and returns tomorrow with continuity intact.

## Current Product Direction

- Public entry is identity and membership.
- Authenticated experience is participation and progression.
- Axis History is the persistent record.
- Leaderboard adds quiet social tension from effort continuity.
- Organizations feel alive through real activity, streaks, sessions, and active-today signals.
- The product should feel like a real athletic continuity operating system, not a prototype or website.

## Out Of Scope

Until the current loop is stable, do not build:

- Media expansion.
- Commentary.
- Feeds.
- Analytics dashboards.
- Speculative topology or runtime systems.
- Replay-first products.
- Speculative AI systems.
- Telemetry experiments.
- Startup SaaS patterns.
