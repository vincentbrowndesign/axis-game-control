# Axis World Model

Axis is a behavioral basketball memory system.

It is not a replay uploader, generic sports analytics product, or AI coaching dashboard.

## Core Objects

- Digital Twin: the persistent memory identity.
- Warmup: daily discipline disguised as memory refinement.
- Replay: memory retrieval.
- Archive: behavioral continuity over time.
- Read: a confidence-gated observation created after replay loads.

## Digital Twin

The Digital Twin is the machine's memory anchor. It connects:

- warmup chains
- replay continuity
- baseline formation
- future comparison reads
- long-term player memory

It is not a social profile, avatar system, or public identity surface.

## Flow

After login:

`ENTER AXIS -> CREATE DIGITAL TWIN -> START MEMORY -> PICK WARMUP -> RECORD WITH AXIS -> MEMORY STORED`

If no twin exists, use `LOCAL PLAYER` as a safe fallback, but authenticated users should be guided into creating a Digital Twin before warmups.
