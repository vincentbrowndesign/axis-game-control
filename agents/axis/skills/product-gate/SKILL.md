# Axis Product Gate Skill

## When to Activate

Before writing any code. Before accepting any task. Before designing any feature.

## The Gate Question

**Does this change improve:**

```
Record → Moment → Film → Share → Return
```

If the answer is no — do not build it.

## The Product Loop

```
show up
→ check in
→ participate
→ history grows
→ leaderboard updates
→ return tomorrow
```

Every surface must serve this loop. If a change introduces something outside it, question whether it belongs.

## The Intelligence Loop

```
Experiment
→ Claim
→ Verdict
→ Observation
→ Outcome
```

Intelligence is ambient. It rides underneath the product loop. It is never the primary surface.

## Surface Hierarchy Check

Every screen must follow:

```
TOP     — identity + presence signal
CENTER  — dominant ritual layer (Check In, active session, participation)
BOTTOM  — history + continuity records
```

If a proposed change breaks this hierarchy, push back.

## Language Check

| Use              | Avoid              |
|------------------|--------------------|
| check in         | analytics          |
| participation    | optimization       |
| session          | AI insights        |
| history          | scouting           |
| continuity       | machine intelligence |
| streak           | tactical breakdown |
| return           | platform           |
| replay memory    | dashboard          |
| effort           | coaching claims    |

## What Not to Build

```
dashboards
scouting analytics
tactical overlays
assistant chat surfaces
coaching claims
commentary systems
clip-editing suites
timeline editing tools
AI announcement UI
startup framing
giant widget stacks
```

## How to Apply This Skill

1. Read the proposed change in plain English
2. Ask: "Which part of the product loop does this serve?"
3. If no clear answer: do not build it
4. If clear answer: proceed and name it in the task description
