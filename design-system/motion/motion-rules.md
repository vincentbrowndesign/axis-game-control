# Axis Motion Rules

Motion should feel alive but calm.

Use motion for:

- successful check-in state changes
- active signal breathing
- progression node activation
- subtle surface focus
- history updates

Do not use motion for:

- decoration
- flashy reward bursts
- constant idle animation
- hover spam
- dashboard theatrics

Timing:

- quick feedback: `var(--axis-motion-fast)`
- normal transitions: `var(--axis-motion)`
- atmospheric changes: `var(--axis-motion-slow)`
