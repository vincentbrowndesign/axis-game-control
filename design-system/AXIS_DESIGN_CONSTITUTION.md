# Axis Design Constitution

Axis is a soft athletic operating system.

The visual system exists to protect the current product loop:

show up -> participate -> progression updates -> history grows -> leaderboard updates -> return tomorrow

Axis should feel like a living sports continuity operating system: warm, active, premium, communal, and calm.

Axis is not a SaaS dashboard, startup site, cyberpunk terminal, analytics surface, replay-first product, speculative AI surface, telemetry experiment, or disconnected set of pages.

## Typography Rules

- Use giant athletic typography for the emotional center of a screen.
- Use clean premium display type for identity, participation, progression, and major state changes.
- Use small monospace metadata for streaks, timestamps, leaderboard rank, and system records.
- Keep copy minimal. Let state, scale, and progression carry meaning.
- Use soft but strong hierarchy: one dominant phrase, a few supporting records.

Avoid:

- hacker fonts
- gimmick futurism
- startup typography
- dense explanatory text
- repeated labels that make the app feel administrative

## Surface Rules

The UI should feel:

- soft
- rounded
- breathable
- premium
- athletic
- alive

Prefer modules that resemble Apple Fitness, PS5 cards, Oura, premium wearable interfaces, and athletic electronics.

Move away from hard terminal rectangles. Axis can keep electronic confidence without becoming cold or brittle.

## Color Rules

The palette should feel warm, active, human, and electrical.

Use:

- warm light concrete
- soft graphite
- arena tunnel wall neutrals
- premium wearable surfaces
- warm graphite
- smoked dark surfaces
- active athletic signal colors
- sharp signal edges
- alive energy

Avoid:

- black-hole darkness
- heavy cinematic shadows
- dead black voids
- lonely cinematic darkness
- cyberpunk neon overload
- rainbow interfaces
- generic startup gradients

Axis can use darkness for contrast, but the default emotional temperature should feel like an active gym during the day: human, warm, and awake. Do not let the interface collapse back into a lonely replay-era void.

Glow must support hierarchy, not replace it. Prefer crisp OLED-like edges, contained active states, and readable typography over broad bloom, blur spread, or foggy light spill.

## Motion Rules

Motion should feel alive but calm.

Use:

- soft transitions
- pulse states
- progression lighting
- active signals
- continuity updates

Avoid:

- flashy animations
- hover spam
- game-like chaos
- blinking urgency
- motion that competes with the ritual

## Screen Rules

Every Axis screen must obey:

- one dominant action
- one emotional state
- few elements
- huge hierarchy
- soft surfaces
- visual progression
- calm structure
- real operational state

Avoid dashboard drift. Do not fill space because space exists.

## Progression Systems

Replace text-heavy continuity with visual systems when the feature is ready.

Core objects:

- streak calendars
- progression grids
- completed-day nodes
- active states
- leaderboard placements
- continuity surfaces

Progression should make the app feel alive through participation, not noisy through decoration.

## Reference Lock

Stable anchors:

- Apple Fitness
- Oura
- PS5 UI
- Xbox cards
- premium wearable interfaces
- athletic operating systems
- soft rounded modules
- progression calendars
- save-state systems

Future screens should feel like they were built from this same system automatically.
