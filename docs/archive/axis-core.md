# Axis Core

This document is the single source of truth for Axis.

All future architecture must align with this document.

## Definition

Axis = Development Architect

Axis issues objectives.

Axis applies constraints.

Humans perform.

Reality responds.

Axis captures context.

Axis remembers progression.

Axis generates the next objective.

## Core Loop

```text
Axis Prompt
→ Objective
→ Constraint
→ Human Action
→ Reality
→ Capture
→ Context
→ Memory
→ Evaluation
→ Next Objective
```

## Signal Stack

```text
Camera = Spatial Context
Microphone = Intent + Continuity
Speaker = Constraint Delivery
Memory = Progression
```

## Build Order

1. Mission Engine
2. Voice
3. Memory
4. Progression
5. Context
6. Camera
7. CV

## Do Not Build

Do not build:

- Replay
- Uploads
- Dashboards
- Scouting
- Skill Trees
- Analytics-first experiences

## Architecture Rule

Every future feature must strengthen the core loop.

If a feature does not support:

```text
Axis Prompt
→ Objective
→ Constraint
→ Human Action
→ Reality
→ Capture
→ Context
→ Memory
→ Evaluation
→ Next Objective
```

it does not belong in Axis.
