# Axis Asset Flywheel

Status:
Strategy

Build Decision:
Define Capsule

Active Runtime:
No

## Core Sentence

Every real-world session can manufacture reusable assets that improve development, produce distribution, and create demand for the next session.

## Canonical Loop

```text
Source Run -> Source Records -> Suggested Structured Records -> Kept Records -> Datasets -> Draft Data Assets -> Output Products -> Distribution Events -> Performance Signals -> Better Next Source Run
```

## Three Loops

### Development Loop

A real session creates source material. Source material can become suggested structure. Useful structure can be kept, grouped, and eventually used to make the next practice, game, meeting, or development session better.

This loop is about improving the work.

### Media Loop

Useful outputs may become recaps, clips, boards, content packages, or other rendered derivatives. Distribution events can show which outputs people reuse, share, question, correct, or ask for again.

This loop is about turning governed work into useful public or semi-public material.

### Business Loop

If repeated field tests show that outputs create reuse and demand, Axis may later support commercial paths around access, development products, team/program tools, media packages, sponsor inventory, or subscriptions.

This loop is about validated demand, not assumed revenue.

## Ads Boundary

Ads are a Distribution Amplifier.

Ads are not the business, not the source of value, and not proof that a data asset is useful. They may later amplify an output product or sponsor inventory, but they do not create verified truth, development value, or demand by themselves.

## Vocabulary Boundaries

- Thread remains the active exact conversation object.
- Research Trail is a possible future derived object.
- Clip is a Source Record, not automatic evidence.
- Kept is a lifecycle state, not a card type.
- Verified requires future Evidence/Witness rules.
- Output Product is derivative, not source of truth.
- Distribution Events are not Data Assets.
- Performance Signals are not verified truth.

## Possible Future Commercial Paths

All paths below are future and unvalidated:

- family access
- player recaps
- training and development products
- team/program tools
- clip and content packages
- sponsor inventory
- subscriptions

## Youth Data Rule

Axis may monetize tools, access, reports, development products, media, and sponsor inventory.

Axis must not monetize raw child data.

## Manual Field-Test Scorecard

Track these by hand before claiming a flywheel:

- source runs completed
- source records captured
- suggested records created
- kept records selected
- output products produced
- output reuse count
- distribution events
- views/shares/comments
- parent/player inquiries
- subscriber or sponsor signals
- correction rate
- unsupported-claim rate
- consent coverage
- turnaround time
- next-session changes caused by the assets

## No Fake Compounding Claims

- do not describe revenue as infinite
- do not assume every source creates valuable outputs
- do not count views as revenue
- do not count suggested records as verified assets
- do not claim a flywheel until repeated field tests show reuse and demand

## Do Not Build

This strategy does not activate:

- operational Data Asset Layer
- database tables
- structured-record persistence
- dataset generation
- Keeper UI
- evidence verification
- asynchronous workers
- reports or exports
- subscriptions
- marketplace
- sponsor tooling
- analytics dashboard
- ad integrations
- Lens/CV ingestion
- cross-thread player memory
- new page
- new mode
