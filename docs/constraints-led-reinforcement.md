# Constraints-Led Reinforcement

Axis uses constraints to help corrections transfer from controlled practice into live basketball.

The coach chooses the constraint. Axis stores it with the clip and makes it retrievable later.

## Principle

Players learn better when the practice environment asks for the same perception and decision the game will ask for.

The goal is not isolated repetition. The goal is a correction that survives timing, spacing, defenders, teammates, and pressure.

## Representative Learning

Practice should keep the important parts of the game problem intact:

- Where the defender is.
- Where the help is.
- Where the pass comes from.
- Where the next decision lives.
- What space is open or closed.
- What the player must see before acting.

Axis should help coaches retrieve clips by those basketball conditions, not just by drill name.

## Environmental Constraints

Common constraints:

- 2 dribbles max.
- Weak hand only.
- No middle drive.
- One-touch finish.
- No retreat dribble.
- Reject screen.
- Automatic kick on slot drive.

Constraints should be short, coach-readable, and connected to the situation.

## Behavioral Transfer

A correction is not finished when it works in a clean rep.

It needs to show up again in:

- Block work.
- Guided pressure.
- Scrimmage.
- Game-speed decisions.

Axis tracks the clip, trigger, constraint, and phase so the coach can reinforce the same behavior across environments.

## Perception-Action Coupling

The cue and the action should stay connected.

Example:

- Situation: High Pick-and-Roll.
- Flaw: Rounded off the screen.
- Correction: Shave the screener's shoulder.
- Trigger: TIGHT.
- Constraint: Reject screen if the defender jumps high.

The player is not just repeating a move. The player is learning what to see and how to respond.
