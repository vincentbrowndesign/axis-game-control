# Coaching Reinforcement

Axis stores the coach's correction in a simple structure:

Flaw -> Correction -> Trigger.

The coach writes it. Axis retrieves it.

## Flaw

The flaw is what went wrong in basketball terms.

Examples:

- Rounded off screen wide.
- Stood tall on catch.
- Picked up dribble before help committed.
- Opened hips too early on closeout.

## Correction

The correction is what the player should do next time.

Examples:

- Shave the screener's shoulder.
- Drop hips before the catch.
- Keep dribble alive until the low man commits.
- Close short, then slide.

## Trigger

The trigger is the short cue used during practice or live play.

Examples:

- TIGHT.
- SINK.
- LOW.
- HOLD.
- HIT.

## Repeat Tomorrow

Repeat status means the coach wants the clip retrieved before the next practice.

Repeat should be used for:

- A correction that must be reinforced.
- A tactical situation that keeps showing up.
- A player who needs the same cue again.
- A team habit that needs another rep under pressure.

## Retrieval Timing

Useful retrieval moments:

- Before practice plan.
- Before a skill block.
- Before scrimmage.
- Before film review.
- Before a game where the same situation is likely.

## Reinforcement Loop

1. Save the clip.
2. Tag the situation.
3. Write the flaw.
4. Write the correction.
5. Assign the trigger.
6. Set the constraint.
7. Mark repeat if needed.
8. Retrieve before the next live rep.
