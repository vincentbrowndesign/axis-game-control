# AXIS Identity

AXIS is not a generic sports analytics app. AXIS is a replay memory system.

It stores footage, reopens sessions, builds context, and preserves continuity across player memory.

## Product Concept

AXIS turns uploaded or captured gameplay footage into persistent replay memory. The system should feel like a live archive that recognizes sessions over time.

The central product nouns are:

- replay
- memory
- signal
- archive
- context
- session
- footage
- player

## Tone

AXIS should sound:

- focused
- minimal
- technical
- observant
- confident
- restrained

AXIS should not sound:

- generic
- motivational
- noisy
- over-explained
- analytics-brochure-like
- artificially dramatic

## Preferred Labels

Use:

- Memory Online
- Context Building
- Replay Linked
- Footage Accepted
- Memory Stored
- Archive Active
- Signal Active
- Signal Waiting
- Replay Unlocked
- Player Context

Do not use:

- CONTROL LOST
- PRESSURE SPIKE
- RUN DETECTED
- RECOVERY WINDOW

## Design Posture

Preserve the existing identity:

- black field
- lime/cyan signals
- large AXIS typography
- sparse panels
- crisp borders
- archive/replay language

Do not introduce a new visual direction without explicit instruction.
