# Trainer Protocol

Axis helps trainers keep corrections consistent across sessions.

The trainer's job is to define the basketball meaning. Axis stores and retrieves it.

## Session Workflow

1. Capture the clip.
2. Tag the situation.
3. Write the flaw.
4. Write the correction.
5. Assign one trigger.
6. Add the constraint.
7. Set Construction Zone status.
8. Set Stress Phase.
9. Mark repeat if it should return tomorrow.

## Trigger Usage

Use the same trigger for the same correction.

Example:

- Situation: High Pick-and-Roll.
- Flaw: Rounded off screen wide.
- Correction: Shave the screener's shoulder.
- Trigger: TIGHT.

The trigger should be usable during a live rep without stopping the workout.

## Constraint Usage

Constraints should shape the behavior.

Examples:

- No middle drive for closeout defense.
- 2 dribbles max for advantage decisions.
- Weak hand only for ball pressure.
- Automatic kick on slot drive.

Do not use constraints as punishment.

Use them to make the player solve the right basketball problem.

## Repeat Structure

Mark repeat when:

- The flaw is recurring.
- The correction needs another touch.
- The same situation will appear next practice.
- The player needs a cue before live pressure.

## Language Standard

Use plain basketball language.

Avoid dystopian language, surveillance language, hype, fake certainty, and abstract product framing.
