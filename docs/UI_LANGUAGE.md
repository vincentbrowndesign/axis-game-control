# UI Language

AXIS has a distinct visual and verbal identity. Preserve it.

## Visual Identity

AXIS UI currently uses:

- black backgrounds
- white text
- lime signal accents
- cyan secondary signal accents
- thin white borders with low opacity
- uppercase micro-labels
- wide tracking on system labels
- heavy compressed display type
- stark replay surfaces
- minimal card ornamentation

Do not redesign the UI unless explicitly asked.

## Voice

AXIS language should feel like a replay memory system:

- direct
- precise
- signal-aware
- calm
- archival
- system-native

Preferred phrases:

- Memory Online
- Context Building
- Replay Linked
- Footage Accepted
- Memory Stored
- Archive Active
- Player Context
- Session Continuity
- Axis Replay System
- Axis Memory Replay

Avoid generic sports analytics language such as:

- performance dashboard
- player efficiency
- heat map insights
- scouting report
- key takeaways
- advanced analytics

Avoid dramatic failure labels:

- CONTROL LOST
- PRESSURE SPIKE
- RUN DETECTED
- RECOVERY WINDOW

## Interaction Language

Use upload and replay words consistently:

- upload
- replay
- footage
- memory
- archive
- signal
- context
- session

Do not replace AXIS terms with generic SaaS labels unless the user explicitly asks.

## UI Change Rule

Do not redesign layout, palette, typography, or interaction model unless explicitly asked.

When changing UI to support a feature:

- Keep the black/lime/cyan AXIS palette.
- Keep uppercase system labels.
- Keep replay memory terminology.
- Avoid marketing-page language.
- Avoid generic sports analytics framing.
