# Interaction Philosophy

Axis interactions should feel like activating a personal device, not clicking a website.

## Ritual Rule

Check In is the current emotional center.

The action should feel:

- Physical.
- Intentional.
- Ceremonial.
- Terminal-like.
- Earned.

The user should feel:

"I kept the chain alive."

## Active Ritual State

Before check-in:

Check in

After check-in:

Checked in - time

Use the same interaction object for both states. Do not create competing buttons, widgets, banners, or status cards.

## Metadata Rule

Streak and last check-in are archival records.

They should feel like:

- Save data.
- Profile continuity.
- Terminal records.
- Persistent identity state.

They should not feel like analytics or dashboard metrics.

## Leaderboard Rule

Leaderboard is social tension, not social media.

Rank only effort continuity:

- Verified check-ins.
- Hours.
- Active streak.
- Monthly consistency.
- Sessions completed.

Never rank talent, points, popularity, likes, comments, or attention.

## Avoid

- Rounded SaaS buttons.
- Bright CTA treatments.
- Gamified badges.
- Charts.
- Feeds.
- Admin panels.
- Productivity workflows.
- Explanatory product copy.

Subtraction is the default refinement move.
