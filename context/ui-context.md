# UI Context

Axis visual direction:

- Identity-driven.
- Calm.
- Ritualistic.
- Persistent.
- Emotionally grounded.
- Structurally stable.
- Intelligence underneath.

Current UI rule:

Do not redesign while stabilizing the identity -> physical presence -> history persistence seam.

Use the existing Axis visual language:

- Black, charcoal, smoke gray, white.
- Subtle transparency.
- Restrained glow only where already established.
- Calm typography.
- Structural layout, not dashboard stacks.
- Public pages should feel like entry, membership, belonging, and presence.
- Internal pages should feel like a persistent personal sports history system.

Preferred product language:

- Axis History.
- Check in.
- History.
- Presence.
- Streak.
- Progression.
- Return tomorrow.

Avoid product language:

- Training continuity.
- Memory system.
- Dashboard.
- Pipeline.
- Telemetry.
- Abstract AI language.

Avoid UI moves:

- Dashboard-heavy surfaces.
- Loud gradients.
- New visual experiments.
- Extra panels for clips, broadcast, overlays, or commentary.
- Replay-first surfaces before the current loop is stable.
