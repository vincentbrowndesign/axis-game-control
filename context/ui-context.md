# UI Context

Axis visual direction:

- Identity-driven.
- Calm.
- Ritualistic.
- Persistent.
- Emotionally grounded.
- Structurally stable.
- Intelligence underneath.
- Personal sports device.
- Save file system.
- Membership terminal.
- 90s/2000s electronics identity.

Current UI rule:

Do not redesign while stabilizing the identity -> physical presence -> history persistence seam.

Use the existing Axis visual language:

- Layered blacks: #000000, #050505, #0b0b0b, #111111, #161616.
- Subtle transparency.
- One restrained hardware accent only: muted cyan, pale phosphor blue, faded amber, or soft CRT green.
- Restrained glow only for active states, focus states, successful ritual completion, or subtle signal moments.
- Calm typography.
- Oversized clean display typography paired with tiny monospace metadata.
- Structural layout, not dashboard stacks.
- Public pages should feel like entry, membership, belonging, and presence.
- Internal pages should feel like a persistent personal sports history system.
- Check In is the emotional center.
- Streak and last check-in should feel archival, like save data or terminal records.

Preferred product language:

- Axis History.
- Check in.
- History.
- Presence.
- Streak.
- Progression.
- Return tomorrow.

Avoid product language:

- Training continuity.
- Memory system.
- Dashboard.
- Pipeline.
- Telemetry.
- Abstract AI language.

Avoid UI moves:

- Dashboard-heavy surfaces.
- Loud gradients.
- New visual experiments.
- Extra panels for clips, broadcast, overlays, or commentary.
- Replay-first surfaces before the current loop is stable.
- Flat black-only surfaces.
- Rainbow accents.
- Startup gradients.
- Generic web buttons.
